Build a complete React Native mobile app called "SplitEasy" using Expo. This should run as a mobile app (Android + iOS). Use Expo Router for navigation and AsyncStorage for local data (no backend needed).

---

## TECH STACK
- Framework: React Native + Expo (SDK 51)
- Navigation: Expo Router (file-based routing)
- Storage: AsyncStorage (all data stored locally on device)
- Styling: StyleSheet + custom theme (no Tailwind)
- Icons: @expo/vector-icons (Ionicons)
- No backend, no database, no login required

## TAB 1 — EXPENSES

- FlatList of expense cards
- Each card shows:
  - Category icon (colored circle with emoji: 🍕 Food, ✈️ Travel, 🏠 Rent, 🎉 Fun, 📦 Other)
  - Expense title + amount (₹)
  - "Paid by [Name]" + date
  - Swipe left to delete (with confirmation alert)
- Floating "+" button (bottom right) → opens Add Expense modal
- Empty state: "No expenses yet. Add your first one!"

### Add Expense Modal (Bottom Sheet style)
- Title input
- Amount input (numeric keyboard, ₹ prefix)
- Category picker (horizontal scrollable chips)
- "Paid by" picker (member list)
- Split type toggle: Equal | Custom
  - Equal: shows each person's share automatically
  - Custom: input field for each member (validates sum = total)
- Date picker (defaults to today)
- "Save Expense" button (green, full width)
- "Cancel" button

---

## TAB 2 — BALANCES

- Summary cards per member:
  - Avatar circle with initials
  - Name
  - "Paid ₹X total"
  - Net balance: green "+₹X (gets back)" or red "-₹X (owes)"
- Divider
- "Simplified Debts" section:
  - Each row: "[Name] owes [Name] ₹X"
  - Arrow icon between names
  - Clean card style

### Balance Calculation Logic:
1. Loop all expenses → record paid_by and each member's owed amount
2. Net balance per person = total_paid - total_owed
3. Greedy simplification:
   - creditors = members with positive balance (sorted desc)
   - debtors = members with negative balance (sorted asc)
   - Match largest debtor to largest creditor, reduce, repeat
4. Return minimal list of "A owes B ₹X"

---

## TAB 3 — SETTLE UP

- List of pending settlements (from balance calculation)
- Each row:
  - From name → To name
  - Amount in green
  - "Mark Paid" button (outline style)
  - "Pay via UPI" button → opens upi://pay?pa=&am=XXX&cu=INR deeplink
- Completed settlements section (grayed out, checkmark icon)
- If all settled: success illustration + "All settled up! 🎉"

## DATA STRUCTURE (AsyncStorage)

Key: "splitease_groups" → array of group objects

Group object:
{
  id: string,
  code: string (6 chars uppercase),
  name: string,
  createdAt: string,
  members: [{ id, name }],
  expenses: [
    {
      id, title, amount, category,
      paidById, date,
      splits: [{ memberId, amountOwed }]
    }
  ],
  settlements: [
    { id, fromId, toId, amount, isPaid }
  ]
}
